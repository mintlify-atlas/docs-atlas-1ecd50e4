title: "Set up your Agropulse development environment"
description: "Get Agropulse running locally in minutes using Poetry and PostgreSQL. Covers Python requirements, environment variables, migrations, and Makefile shortcuts."
---

Agropulse requires Python 3.12 or newer, PostgreSQL, and Poetry. This guide walks through every step from cloning the repository to starting the development server, including all environment variables you need to configure.

<Steps>
  <Step title="Install Python 3.12+ and Poetry">
    Confirm your Python version before proceeding:

    ```bash
    python --version   # must be 3.12 or newer
    ```

    Install Poetry if you do not already have it:

    ```bash
    curl -sSL https://install.python-poetry.org | python3 -
    ```

    Verify the installation:

    ```bash
    poetry --version
    ```
  </Step>

  <Step title="Clone the repository and install dependencies">
    ```bash
    git clone https://github.com/OluwagbeminiyiA/agro_pulse-API.git
    cd agro_pulse-API
    poetry install
    ```

    This installs the full dependency set including Django 6, Django REST Framework, SimpleJWT, django-filter, django-otp, drf-spectacular, Celery, Redis, and the Squad payment SDK wrapper.
  </Step>

  <Step title="Create your .env file">
    Create a `.env` file in the project root. All variables below are required unless marked optional.

    ```bash
    # Django core
    DJANGO_SECRET_KEY=your-secret-key-here
    DEBUG=True
    ALLOWED_HOSTS=localhost,127.0.0.1
    CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000

    # PostgreSQL database
    DB_NAME=agropulse
    DB_USER=postgres
    DB_PASSWORD=your-db-password
    DB_HOST=localhost

    # Email / OTP (uses Gmail SMTP by default)
    EMAIL_SENDER=you@gmail.com
    EMAIL_PASSWORD=your-gmail-app-password
    OTP_EMAIL_THROTTLE_FACTOR=1
    OTP_EMAIL_COOLDOWN_DURATION=60
    OTP_EMAIL_TOKEN_VALIDITY=300

    # Squad payment gateway
    SQUAD_SECRET_KEY=sk_sandbox_...
    SQUAD_PUBLIC_KEY=pk_sandbox_...
    SQUAD_MERCHANT_ID=your-merchant-id
    SQUAD_BASE_URL=https://sandbox-api-d.squadco.com
    SQUAD_PAYMENT_URL=https://sandbox-pay.squadco.com
    ```

    <Note>
      `EMAIL_PASSWORD` should be a Gmail App Password, not your account password. Generate one at **Google Account → Security → App Passwords**.
    </Note>
  </Step>

  <Step title="Create the PostgreSQL database">
    ```bash
    psql -U postgres -c "CREATE DATABASE agropulse;"
    ```

    Agropulse is PostgreSQL-only; SQLite is not a supported runtime database.
  </Step>

  <Step title="Run migrations">
    ```bash
    poetry run python -m core_agropulse.manage migrate
    ```
  </Step>

  <Step title="Create a superuser">
    ```bash
    poetry run python -m core_agropulse.manage createsuperuser
    ```

    The superuser account gives you access to the Django admin interface at `/admin/` and grants staff-level API permissions such as viewing all payments and processing payouts.
  </Step>

  <Step title="Start the development server">
    ```bash
    poetry run python -m core_agropulse.manage runserver
    ```

    The API is now available at `http://localhost:8000/api/`. The interactive Swagger UI is served at `/api/schema/swagger-ui/`.
  </Step>
</Steps>

## Environment variable reference

| Variable | Required | Description |
|---|---|---|
| `DJANGO_SECRET_KEY` | Yes | Django cryptographic secret. Generate with `python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"` |
| `DEBUG` | Yes | Set `True` for local development, `False` in production |
| `ALLOWED_HOSTS` | Yes | Comma-separated list of allowed hostnames |
| `CORS_ALLOWED_ORIGINS` | Yes | Comma-separated list of allowed CORS origins |
| `DB_NAME` | Yes | PostgreSQL database name |
| `DB_USER` | Yes | PostgreSQL user |
| `DB_PASSWORD` | Yes | PostgreSQL password |
| `DB_HOST` | Yes | PostgreSQL host (e.g. `localhost`) |
| `EMAIL_SENDER` | Yes | Gmail address used to send OTP emails |
| `EMAIL_PASSWORD` | Yes | Gmail App Password |
| `OTP_EMAIL_THROTTLE_FACTOR` | Yes | Multiplier applied to OTP throttle intervals |
| `OTP_EMAIL_COOLDOWN_DURATION` | Yes | Seconds a user must wait between OTP requests |
| `OTP_EMAIL_TOKEN_VALIDITY` | Yes | Seconds an OTP token remains valid |
| `SQUAD_SECRET_KEY` | Yes | Squad secret key (prefixed `sk_`) |
| `SQUAD_PUBLIC_KEY` | Yes | Squad public key (prefixed `pk_`) |
| `SQUAD_MERCHANT_ID` | Yes | Your Squad merchant ID |
| `SQUAD_BASE_URL` | Yes | Squad API base URL |
| `SQUAD_PAYMENT_URL` | Yes | Squad checkout URL |

## Makefile commands

The repository ships a Makefile with the most common development tasks:

| Command | What it runs |
|---|---|
| `make run-server` | `poetry run python -m core_agropulse.manage runserver` |
| `make migrations` | `poetry run python -m core_agropulse.manage makemigrations` |
| `make migrate` | `poetry run python -m core_agropulse.manage migrate` |
| `make lint` | `poetry run pre-commit run --all-files` (uses Ruff) |
| `make test` | Runs the test suite |

## Running the test suite

Run all tests:

```bash
poetry run python -m core_agropulse.manage test
```

Run tests for a specific app with verbose output:

```bash
poetry run python -m core_agropulse.manage test core_agropulse.payments.tests -v 2
poetry run python -m core_agropulse.manage test core_agropulse.subscriptions.tests -v 2
```
