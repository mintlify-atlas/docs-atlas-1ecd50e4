---
title: "Subscriptions API — recurring produce orders"
description: "Endpoints for creating and managing recurring produce subscriptions, auto-generated subscription orders, and associated payment records for buyers and farmers."
---

The Subscriptions API lets buyers set up recurring produce orders from a specific farmer at a chosen frequency. Once active, the system auto-generates `SubscriptionOrder` records on each scheduled date. Payments are tracked separately via `SubscriptionPayment` records linked one-to-one to each subscription order.

<Note>
  A buyer can hold only one active subscription per farmer–produce combination. Attempting to create a duplicate returns a validation error.
</Note>

## Access control

Results are automatically scoped by role:

- **Admin** — all subscriptions, orders, and payments.
- **Buyer** — only their own subscriptions, and orders and payments derived from them.
- **Farmer** — subscriptions, orders, and payments associated with their produce.

## Subscriptions

### List subscriptions

<ParamField path="GET /api/subscriptions/" type="string">
  Returns a list of subscriptions visible to the authenticated user.
</ParamField>

**Query parameters**

<ParamField query="status" type="string">
  Filter by subscription status. One of `ACTIVE`, `PAUSED`, `CANCELLED`.
</ParamField>

<ParamField query="frequency" type="string">
  Filter by delivery frequency. One of `DAILY`, `WEEKLY`, `MONTHLY`.
</ParamField>

<ParamField query="active" type="boolean">
  Filter by active flag. `true` returns only active subscriptions; `false` returns paused or cancelled.
</ParamField>

<ParamField query="ordering" type="string">
  Sort by `created_at` or `next_expected_order_date`. Prefix with `-` for descending. Defaults to `-created_at`.
</ParamField>

<CodeGroup>

```bash List subscriptions
curl --request GET \
  --url https://api.agropulse.example.com/api/subscriptions/ \
  --header 'Authorization: Bearer <token>'
```

```bash Filter active weekly subscriptions
curl --request GET \
  --url 'https://api.agropulse.example.com/api/subscriptions/?status=ACTIVE&frequency=WEEKLY' \
  --header 'Authorization: Bearer <token>'
```

</CodeGroup>

### Create a subscription

<ParamField path="POST /api/subscriptions/" type="string">
  Creates a new recurring subscription for a buyer–farmer–produce combination.
</ParamField>

<Warning>
  The `buyer + farmer + produce` combination must be unique. Creating a duplicate returns a 400 error.
</Warning>

**Request body**

<ParamField body="buyer" type="string" required>
  UUID of the buyer profile placing the subscription.
</ParamField>

<ParamField body="farmer" type="string" required>
  UUID of the farmer profile supplying the produce.
</ParamField>

<ParamField body="produce" type="string" required>
  UUID of the produce item to subscribe to.
</ParamField>

<ParamField body="frequency" type="string" required>
  Delivery cadence. One of `DAILY`, `WEEKLY`, `MONTHLY`.
</ParamField>

<ParamField body="expected_quantity" type="number" default="1">
  Number of units expected each cycle.
</ParamField>

<ParamField body="next_expected_order_date" type="string" required>
  ISO 8601 date for the first order. Must be today or in the future.
</ParamField>

<CodeGroup>

```bash Create subscription
curl --request POST \
  --url https://api.agropulse.example.com/api/subscriptions/ \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "buyer": "7c9e6679-7425-40de-944b-e07fc1f90ae7",
    "farmer": "3d6f4445-eb35-4b54-99f9-a2f4cf0b1234",
    "produce": "b5e4a3c2-1d0e-4f8b-9a7c-6e5d4c3b2a1f",
    "frequency": "WEEKLY",
    "expected_quantity": 5,
    "next_expected_order_date": "2024-11-08"
  }'
```

```json 201 Response

  "produce_price": "500.00",
  "frequency": "WEEKLY",
  "expected_quantity": 5,
  "next_expected_order_date": "2024-11-08",
  "active": true,
  "status": "ACTIVE",
  "subscription_start_date": "2024-11-01",
```

### Update subscription

<ParamField path="PATCH /api/subscriptions/{id}/" type="string">
  Partially updates a subscription. You can change `frequency`, `expected_quantity`, `next_expected_order_date`, or `status` directly, but prefer the dedicated action endpoints for pause, resume, and cancel operations.
</ParamField>

**Path parameters**

<ParamField path="id" type="string" required>
  UUID of the subscription.
</ParamField>

**Request body (all fields optional)**

<ParamField body="frequency" type="string">
  New delivery frequency. One of `DAILY`, `WEEKLY`, `MONTHLY`.
</ParamField>

<ParamField body="expected_quantity" type="number">
  Updated quantity per cycle.
</ParamField>

<ParamField body="next_expected_order_date" type="string">
  Reschedule the next order date (ISO 8601 date).
</ParamField>

**Pause, resume, and cancel actions**

Use these POST endpoints to transition subscription status. They update both `status` and `active` atomically, and record `subscription_end_date` on cancellation.

| Action | Endpoint | Effect |
|--------|----------|--------|
| Pause | `POST /api/subscriptions/{id}/pause/` | Sets `status` to `PAUSED`, `active` to `false`. |
| Resume | `POST /api/subscriptions/{id}/resume/` | Sets `status` to `ACTIVE`, `active` to `true`. |
| Cancel | `POST /api/subscriptions/{id}/cancel/` | Sets `status` to `CANCELLED`, `active` to `false`, records `subscription_end_date`. |

<CodeGroup>

```bash Pause subscription
curl --request POST \
  --url https://api.agropulse.example.com/api/subscriptions/a1b2c3d4-e5f6-7890-abcd-ef1234567890/pause/ \
  --header 'Authorization: Bearer <token>'
```

```bash Resume subscription
curl --request POST \
  --url https://api.agropulse.example.com/api/subscriptions/a1b2c3d4-e5f6-7890-abcd-ef1234567890/resume/ \
  --header 'Authorization: Bearer <token>'
```

```bash Cancel subscription
curl --request POST \
  --url https://api.agropulse.example.com/api/subscriptions/a1b2c3d4-e5f6-7890-abcd-ef1234567890/cancel/ \
  --header 'Authorization: Bearer <token>'
```

</CodeGroup>

<Note>
  Cancellation sets `subscription_end_date` to today's date and stops future order generation. A cancelled subscription cannot be resumed.
</Note>

## Subscription orders

Subscription orders are auto-generated by the system on each scheduled date. This resource is read-only — you cannot create or delete subscription orders via the API.

### List subscription orders

<ParamField path="GET /api/subscription-orders/" type="string">
  Returns a list of subscription orders visible to the authenticated user.
</ParamField>

**Query parameters**

<ParamField query="subscription" type="string">
  Filter by subscription UUID.
</ParamField>

<ParamField query="order_status" type="string">
  Filter by order status. One of `PENDING`, `CONFIRMED`, `DELIVERED`, `FAILED`, `SKIPPED`.
</ParamField>

<ParamField query="ordering" type="string">
  Sort by `scheduled_date` or `created_at`. Prefix with `-` for descending. Defaults to `-scheduled_date`.
</ParamField>

<CodeGroup>

```bash List subscription orders
curl --request GET \
  --url https://api.agropulse.example.com/api/subscription-orders/ \
  --header 'Authorization: Bearer <token>'
```

```bash Filter pending orders for a subscription
curl --request GET \
  --url 'https://api.agropulse.example.com/api/subscription-orders/?subscription=a1b2c3d4-e5f6-7890-abcd-ef1234567890&order_status=PENDING' \
  --header 'Authorization: Bearer <token>'
```

</CodeGroup>

**Response fields**

<ResponseField name="id" type="string" required>
  UUID of the subscription order.
</ResponseField>

<ResponseField name="subscription" type="string" required>
  UUID of the parent subscription.

<ResponseField name="order_status" type="string" required>

<ResponseField name="scheduled_date" type="string" required>

<ResponseField name="created_at" type="string">
  ISO 8601 datetime the record was created. Read-only.

<ResponseField name="updated_at" type="string">
