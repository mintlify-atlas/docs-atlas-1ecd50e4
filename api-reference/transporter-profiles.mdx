---
title: "Transporter Profiles API — rider setup"
description: "Create and manage transporter profiles in Agropulse. Store vehicle type, plate number, and service area for riders fulfilling produce deliveries."
---

The Transporter Profiles API allows users with the `TRANSPORTER` role to register their vehicle and coverage details. Each profile captures the vehicle type, plate number, and the geographic area the transporter serves. Profiles are scoped to the authenticated user — regular users can only see and modify their own profile; staff can access all profiles.

<Note>
  All endpoints require authentication. Include `Authorization: Bearer <access_token>` in every request.
</Note>

---

## GET /api/transporters/

List transporter profiles. Regular users receive only their own profile. Staff users receive all profiles in the system.

### Response — 200 OK

Returns a JSON array of transporter profile objects.

<ResponseField name="id" type="string">
  UUID of the transporter profile.
</ResponseField>

<ResponseField name="user" type="string">
  UUID of the linked user account (read-only).
</ResponseField>

<ResponseField name="user_email" type="string">
  Email address of the linked user (read-only).
</ResponseField>

<ResponseField name="user_full_name" type="string">
  Full name of the linked user (read-only).
</ResponseField>

<ResponseField name="vehicle_type" type="string">
  Description of the vehicle, e.g. `"Motorcycle"`, `"Pickup Truck"`.
</ResponseField>

<ResponseField name="plate_number" type="string">
  Vehicle registration plate number.
</ResponseField>

<ResponseField name="service_area" type="string">
  Geographic area or region the transporter covers.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of profile creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last update.
</ResponseField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/transporters/ \
  --header 'Authorization: Bearer <access_token>'
```

## POST /api/transporters/

Create a new transporter profile for the authenticated user. The `user` field is set automatically; do not include it in the request body.

### Request body

<ParamField body="vehicle_type" type="string" required>
  Type of vehicle used for deliveries, up to 255 characters. For example: `"Motorcycle"`, `"Van"`, `"Pickup Truck"`.
</ParamField>

<ParamField body="plate_number" type="string" required>
  Vehicle registration plate number, up to 50 characters.
</ParamField>

<ParamField body="service_area" type="string" required>
  Geographic area or region the transporter covers, up to 255 characters.
</ParamField>

### Response — 201 Created

Returns the newly created transporter profile object.

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.agropulse.example.com/api/transporters/ \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "vehicle_type": "Motorcycle",
    "plate_number": "ABC-123-XY",
    "service_area": "Ibadan, Oyo State"
  }'
```

## GET /api/transporters/{id}/

Retrieve a single transporter profile by UUID. Regular users can only access their own profile.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the transporter profile.
</ParamField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/transporters/d4e5f6a7-b8c9-0123-defa-234567890123/ \
  --header 'Authorization: Bearer <access_token>'
```

## DELETE /api/transporters/{id}/

Delete a transporter profile by UUID. Returns `204 No Content` on success. The linked user account is not affected.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the transporter profile to delete.
</ParamField>

<Warning>
  Deleting a transporter profile is permanent. Any active delivery assignments linked to this profile may be disrupted. The linked user account is retained.
</Warning>

```bash cURL
curl --request DELETE \
  --url https://api.agropulse.example.com/api/transporters/d4e5f6a7-b8c9-0123-defa-234567890123/ \
  --header 'Authorization: Bearer <access_token>'
```
