---
title: "Buyer Profiles API — buyer account setup"
description: "Create and manage buyer profiles in Agropulse. Supports individual, restaurant, and wholesaler buyer types with optional business name and required location."
---

The Buyer Profiles API allows users with the `BUYER` role to set up and manage their purchasing identity on Agropulse. Each profile captures the buyer's type, location, and optional business name. Profiles are scoped to the authenticated user — you can only read and modify your own profile unless you are a staff member.

<Note>
  All endpoints require authentication. Include `Authorization: Bearer <access_token>` in every request.
</Note>

---

## GET /api/buyers/

List buyer profiles. Regular users receive only their own profile. Staff users receive all profiles in the system.

### Response — 200 OK

Returns a JSON array of buyer profile objects.

<ResponseField name="id" type="string">
  UUID of the buyer profile.
</ResponseField>

<ResponseField name="user" type="string">
  UUID of the linked user account (read-only).
</ResponseField>

<ResponseField name="user_email" type="string">
  Email address of the linked user (read-only).
</ResponseField>

<ResponseField name="user_full_name" type="string">
  Full name of the linked user (read-only).
</ResponseField>

<ResponseField name="business_name" type="string | null">
  Optional business or organisation name, up to 255 characters.
</ResponseField>

<ResponseField name="buyer_type" type="string">
  Category of buyer. One of: `INDIVIDUAL`, `RESTAURANT`, `WHOLESALER`.
</ResponseField>

<ResponseField name="location" type="string">
  Location or delivery address of the buyer.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of profile creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last update.
</ResponseField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/buyers/ \
  --header 'Authorization: Bearer <access_token>'
```

## POST /api/buyers/

Create a new buyer profile for the authenticated user. The `user` field is set automatically; do not include it in the request body.

### Request body

<ParamField body="buyer_type" type="string" required>
  Category of buyer. Must be one of: `INDIVIDUAL`, `RESTAURANT`, `WHOLESALER`.
</ParamField>

<ParamField body="location" type="string" required>
  Location or delivery address of the buyer, up to 255 characters.
</ParamField>

<ParamField body="business_name" type="string">
  Optional business or organisation name. Leave blank for individual buyers.
</ParamField>

### Response — 201 Created

Returns the newly created buyer profile object.

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.agropulse.example.com/api/buyers/ \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "buyer_type": "RESTAURANT",
    "location": "Victoria Island, Lagos",
    "business_name": "Chidi'\''s Kitchen"
  }'
```


## GET /api/buyers/{id}/

Retrieve a single buyer profile by UUID. Regular users can only access their own profile.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the buyer profile.
</ParamField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/buyers/c3d4e5f6-a7b8-9012-cdef-123456789012/ \
  --header 'Authorization: Bearer <access_token>'
```

## PATCH /api/buyers/{id}/

Partially update a buyer profile. Send only the fields you want to change.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the buyer profile to update.
</ParamField>

### Request body (all fields optional)

<ParamField body="business_name" type="string">
  Updated business name.
</ParamField>

<ParamField body="buyer_type" type="string">
  Updated buyer category. One of: `INDIVIDUAL`, `RESTAURANT`, `WHOLESALER`.
</ParamField>

<ParamField body="location" type="string">
  Updated location or delivery address.
</ParamField>

<CodeGroup>

## DELETE /api/buyers/{id}/

Delete a buyer profile by UUID. Returns `204 No Content` on success. The linked user account is not affected.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the buyer profile to delete.
</ParamField>

<Warning>
  Deleting a buyer profile is permanent. Any associated order history may be affected. The linked user account is retained.
</Warning>

```bash cURL
curl --request DELETE \
  --url https://api.agropulse.example.com/api/buyers/c3d4e5f6-a7b8-9012-cdef-123456789012/ \
  --header 'Authorization: Bearer <access_token>'
```
