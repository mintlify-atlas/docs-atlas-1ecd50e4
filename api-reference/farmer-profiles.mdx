---
title: "Farmer Profiles API — farm management"
description: "Create and manage farmer profiles in Agropulse. Each profile stores farm name, location, and a read-only trust score tied to the authenticated user."
---

The Farmer Profiles API lets users with the `SELLER` role create and manage their farm details. Each profile is linked one-to-one with a user account. When a profile is created, the `user` field is automatically set to the authenticated requester — you never pass it explicitly. Staff members can view all profiles; regular users see only their own.

<Note>
  All endpoints require authentication. Include `Authorization: Bearer <access_token>` in every request.
</Note>

---

## GET /api/farmers/

List farmer profiles. Regular users receive only their own profile. Staff users receive all profiles in the system.

### Response — 200 OK

Returns a JSON array of farmer profile objects.

<ResponseField name="id" type="string">
  UUID of the farmer profile.
</ResponseField>

<ResponseField name="user" type="string">
  UUID of the linked user account (read-only).
</ResponseField>

<ResponseField name="user_email" type="string">
  Email address of the linked user (read-only).
</ResponseField>

<ResponseField name="user_full_name" type="string">
  Full name of the linked user (read-only).
</ResponseField>

<ResponseField name="farm_name" type="string">
  Name of the farm.
</ResponseField>

<ResponseField name="farm_location" type="string">
  Physical location or address of the farm.
</ResponseField>

<ResponseField name="trust_score" type="string">
  Decimal trust score between 0.00 and 9.99, assigned by the platform (read-only).
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of profile creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last update.
</ResponseField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/farmers/ \
  --header 'Authorization: Bearer <access_token>'
```

## POST /api/farmers/

Create a new farmer profile for the authenticated user. The `user` field is set automatically; do not include it in the request body.

### Request body

<ParamField body="farm_name" type="string" required>
  Name of the farm, up to 255 characters.
</ParamField>

<ParamField body="farm_location" type="string" required>
  Physical location or address of the farm, up to 255 characters.
</ParamField>

### Response — 201 Created

Returns the newly created farmer profile object.

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.agropulse.example.com/api/farmers/ \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "farm_name": "Green Valley Farm",
    "farm_location": "Kaduna, Nigeria"
  }'
```

```json Response

  "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",

  "user_email": "ada@example.com",
  "user_full_name": "Ada Okafor",
  "farm_name": "Green Valley Farm",
  "farm_location": "Kaduna, Nigeria",

  "created_at": "2024-01-15T11:00:00Z",

```

## GET /api/farmers/{id}/

Retrieve a single farmer profile by UUID. Regular users can only access their own profile.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the farmer profile.
</ParamField>

```bash cURL
curl --request GET \
  --url https://api.agropulse.example.com/api/farmers/b2c3d4e5-f6a7-8901-bcde-f12345678901/ \
  --header 'Authorization: Bearer <access_token>'
```

## PATCH /api/farmers/{id}/

Partially update a farmer profile. Send only the fields you want to change. `trust_score` is read-only and cannot be set via the API.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the farmer profile to update.
</ParamField>

### Request body (all fields optional)

<ParamField body="farm_name" type="string">
  Updated farm name.
</ParamField>

<ParamField body="farm_location" type="string">
  Updated farm location.
</ParamField>

<CodeGroup>

## DELETE /api/farmers/{id}/

Delete a farmer profile by UUID. Returns `204 No Content` on success. The linked user account is not affected.

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the farmer profile to delete.
</ParamField>

<Warning>
  Deleting a farmer profile is permanent. The associated user account is retained, but the farm details cannot be recovered.
</Warning>

```bash cURL
curl --request DELETE \
  --url https://api.agropulse.example.com/api/farmers/b2c3d4e5-f6a7-8901-bcde-f12345678901/ \
  --header 'Authorization: Bearer <access_token>'
```
