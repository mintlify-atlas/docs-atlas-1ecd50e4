---
title: "Payment flow, escrow, and splits"
description: "Agropulse holds buyer payments in escrow until delivery is confirmed, then automatically splits funds between the farmer (80%), rider (10%), and platform (10%)."
---

Agropulse uses an escrow model to protect both buyers and farmers. When a buyer pays for an order, the money is not sent directly to the farmer—it is held in an escrow account until the buyer confirms receipt of the goods. Once delivery is confirmed, the held amount is split and paid out to the farmer and the transporter, with a platform fee deducted. This design ensures farmers are paid reliably and buyers are protected if a delivery fails.

## Payment channels

Buyers can pay through any of the following channels, all processed via Squad:

| Channel | Description |
|---|---|
| `CARD` | Debit or credit card payment |
| `USSD` | Bank USSD short code |
| `TRANSFER` | Direct bank transfer |
| `VIRTUAL-ACCOUNT` | Payment into a dedicated virtual account number |

Payments are denominated in Nigerian Naira (`NGN`) by default.

## Payment statuses

| Status | Meaning |
|---|---|
| `PENDING` | Payment initiated; awaiting confirmation from Squad |
| `SUCCESS` | Payment confirmed via webhook |
| `FAILED` | Payment attempt failed |
| `REFUNDED` | Funds returned to the buyer |

## The full money flow

<Steps>
  <Step title="Buyer initiates payment">
    The buyer submits an order and is directed to a checkout URL generated by Squad. The `Payment` record is created with `payment_status: PENDING` and `escrow_enabled: true`.
  </Step>
  <Step title="Squad confirms payment via webhook">
    Squad sends a webhook to the Agropulse API. The API verifies the webhook signature, updates `payment_status` to `SUCCESS`, and advances the order to `PAID`.
  </Step>
  <Step title="Escrow holds the funds">
    An `EscrowAccount` record is created, linked to the payment, with `release_status: HELD`. The full payment amount is recorded as `amount_held`. Funds remain here until delivery is confirmed.
  </Step>
  <Step title="Order is fulfilled and delivered">
    The farmer prepares the order and a transporter (if applicable) delivers it. The order moves through `PROCESSING` → `IN_TRANSIT` as each party updates their status.
  </Step>
  <Step title="Delivery confirmed—funds released">
    When the transporter marks the delivery as `DELIVERED`, the order reaches `COMPLETED` and the escrow `release_status` changes from `HELD` to `RELEASED`, recording a `released_at` timestamp.
  </Step>
  <Step title="Payment split calculated">
    A `PaymentSplit` record is created (or updated) for the payment, recording how the total amount is divided:
    - `farmer_amount` — the portion paid to the farmer
    - `rider_amount` — the transporter's delivery fee
    - `platform_fee` — Agropulse's cut

    For `PICKUP` orders where no transporter is involved, `rider_amount` is `0.00`.
  </Step>
  <Step title="Payouts sent">
    Individual `Payout` records are created for the farmer and the rider. Each payout moves through its own status lifecycle (`PENDING` → `PROCESSING` → `COMPLETED`) and is sent to the recipient's virtual account. A `bank_reference` is recorded once the transfer is initiated.
  </Step>
</Steps>

## Escrow account

The `EscrowAccount` model holds the payment between order confirmation and delivery.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Auto-generated, read-only |
| `payment` | Payment | One-to-one link to the associated payment |
| `farmer` | FarmerProfile | The farmer who will receive funds on release |
| `amount_held` | decimal | Total amount held in escrow |
| `release_status` | enum | `HELD`, `RELEASED`, or `DISPUTED` |
| `released_at` | datetime | Set when funds are released; null until then |

```json
{
  "id": "f6a7b8c9-...",
  "payment": "a7b8c9d0-...",
  "farmer": "a1b2c3d4-...",
  "amount_held": "4500.00",
  "release_status": "HELD",
  "released_at": null
}
```

<Warning>
  If a dispute is raised, `release_status` is set to `DISPUTED` and funds remain frozen until the dispute is resolved manually. Disputed escrow accounts do not automatically release.
</Warning>

## Payment split

The `PaymentSplit` model records exactly how a payment is divided once escrow is released.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Auto-generated, read-only |
| `payment` | Payment | One-to-one link to the associated payment |
| `farmer_amount` | decimal | Amount allocated to the farmer |
| `rider_amount` | decimal | Amount allocated to the transporter |
| `platform_fee` | decimal | Amount retained by Agropulse |
| `farmer_processed` | boolean | `true` once the farmer payout is initiated |
| `rider_processed` | boolean | `true` once the rider payout is initiated |

```json
{
  "id": "b8c9d0e1-...",
  "payment": "a7b8c9d0-...",
  "farmer_amount": "4050.00",
  "rider_amount": "300.00",
  "platform_fee": "150.00",
  "farmer_processed": false,
  "rider_processed": false
}
```

## Payouts

Each `Payout` tracks a single disbursement to a farmer or transporter.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Auto-generated, read-only |
| `payment` | Payment | The source payment |
| `farmer` | FarmerProfile | Set for farmer payouts; null for rider payouts |
| `rider` | TransporterProfile | Set for rider payouts; null for farmer payouts |
| `payout_type` | enum | `FARMER` or `RIDER` |
| `amount` | decimal | Amount being paid out |
| `payout_status` | enum | `PENDING`, `PROCESSING`, `COMPLETED`, or `FAILED` |
| `bank_reference` | string | Transfer reference from the payment provider |
| `completed_at` | datetime | Set when the payout reaches `COMPLETED` |

| Payout status | Meaning |
|---|---|
| `PENDING` | Payout queued; not yet initiated |
| `PROCESSING` | Transfer initiated with the bank |
| `COMPLETED` | Funds confirmed as received |
| `FAILED` | Transfer attempt failed; may be retried |

## Virtual accounts

Farmers and transporters each have a `VirtualAccount` used to receive payouts. A virtual account belongs exclusively to either a farmer or a transporter—never both.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Auto-generated, read-only |
| `farmer` | FarmerProfile | Set for farmer accounts |
| `transporter` | TransporterProfile | Set for transporter accounts |
| `virtual_account_number` | string | Unique account number |
| `bank_name` | string | The bank holding the virtual account |
| `account_name` | string | Registered name on the account |
| `is_active` | boolean | Whether the account can receive funds |

<Tip>
  Virtual accounts are created through Squad's virtual account provisioning. Each farmer and transporter should have an active virtual account before their first payout is attempted.
</Tip>
