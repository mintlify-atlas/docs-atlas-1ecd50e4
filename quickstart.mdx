---
title: "Get started with Agropulse API"
description: "Go from zero to your first authenticated API call in minutes — install dependencies, run migrations, start the server, and register a user."
---

Agropulse is a Django REST Framework API. This guide walks you through setting up a local development environment, registering a user, and obtaining a JWT token so you can start making authenticated requests.

<Steps>
  <Step title="Install dependencies">
    Agropulse uses [Poetry](https://python-poetry.org/) for dependency management. Make sure you have Python 3.12+ and Poetry installed, then run:

    ```bash
    poetry install
    ```

    This installs Django, Django REST Framework, SimpleJWT, django-otp, and all other required packages.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root (or export variables into your shell). At minimum you need database credentials and a Django secret key:

    ```bash
    DJANGO_SECRET_KEY=your-django-secret-key
    DEBUG=True
    ALLOWED_HOSTS=localhost,127.0.0.1
    DB_NAME=agropulse
    DB_USER=postgres
    DB_PASSWORD=your-db-password
    DB_HOST=localhost
    ```

    Payments and email OTP require additional Squad and email variables. See the [environment setup guide](/guides/environment-setup) for the full list.
  </Step>

  <Step title="Run database migrations">
    Apply all migrations to create the database schema:

    ```bash
    poetry run python -m core_agropulse.manage migrate
    ```

    Optionally create a superuser for Django admin access:

    ```bash
    poetry run python -m core_agropulse.manage createsuperuser
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    poetry run python -m core_agropulse.manage runserver
    ```

    The API is now available at `http://localhost:8000`. The interactive Swagger UI is at `http://localhost:8000/api/schema/swagger-ui/`.
  </Step>

  <Step title="Register a user">
    Create a new user by posting to `POST /api/users/register/`. You must choose a `role` of `BUYER`, `SELLER`, or `TRANSPORTER`.

    ```bash
    curl -X POST http://localhost:8000/api/users/register/ \
      -H "Content-Type: application/json" \
      -d '{
        "full_name": "Ada Okafor",
        "email": "ada@example.com",
        "phone_number": "+2348012345678",
        "role": "BUYER",
        "password": "strongpassword123",
        "password_confirm": "strongpassword123"
      }'
    ```

    A successful response returns `201 Created` with the new user object:

    ```json
    {
      "id": "3f7e1b2a-...",
      "full_name": "Ada Okafor",
      "email": "ada@example.com",
      "phone_number": "+2348012345678",
      "role": "BUYER",
      "is_active": true,
      "farmer_profile": null,
      "buyer_profile": null,
      "transporter_profile": null,
      "created_at": "2026-05-13T10:00:00Z",
      "updated_at": "2026-05-13T10:00:00Z"
    }
    ```
  </Step>

  <Step title="Log in and get your JWT token">
    Exchange your credentials for a JWT access token and refresh token:

    ```bash
    curl -X POST http://localhost:8000/api/users/login/ \
      -H "Content-Type: application/json" \
      -d '{
        "email": "ada@example.com",
        "password": "strongpassword123"
      }'
    ```

    The response includes both tokens and the full user object:

    ```json
    {
      "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": "3f7e1b2a-...",
        "full_name": "Ada Okafor",
        "email": "ada@example.com",
        "phone_number": "+2348012345678",
        "role": "BUYER",
        "is_active": true,
        "farmer_profile": null,
        "buyer_profile": null,
        "transporter_profile": null,
        "created_at": "2026-05-13T10:00:00Z",
        "updated_at": "2026-05-13T10:00:00Z"
      }
    }
    ```

    Save the `access` token — you'll pass it as a `Bearer` token in the `Authorization` header for every subsequent request.
  </Step>

  <Step title="Make an authenticated request">
    Use your access token to fetch your own profile:

    ```bash
    curl http://localhost:8000/api/users/me/ \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    ```
  </Step>
</Steps>

<Note>
  Access tokens are short-lived. When one expires, use `POST /api/token/refresh/` with your `refresh` token to obtain a new access token without re-entering credentials. See the [authentication guide](/authentication) for details.
</Note>
